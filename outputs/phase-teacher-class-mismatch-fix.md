# "背景跟前景脱节"问题：同标签类别的整体外观差过大时压制强度 + C2 数据重跑

**日期**：2026-07-10（同一天的第三个 teacher 修复，紧接过冲光晕修复 + neutral 加法mood-cast）
**触发**：用户测试一张商场/机场密集人群照片（钢架顶棚 + KING LOMO 招牌），反馈"背景跟前面人物严重脱节"，
质疑既然区分不了就该整图仿色，并要求回顾昨天总结的 Polarr 网页仿色分析。

## 先回顾 Polarr 分析：整图仿色不是没试过，是踩过真坑退回来的

`09-行业方案与知识库/App/2026-07-09-Polarr-ColorMatch仿色分析.md` 里记录着：本项目最早就是从全局 Lab
Reinhard 统计匹配起步的（`scripts/color_transfer.py`），**实测踩过"串色"大坑**——参考图如果有大片强色
（比如红色舞台灯），全局统计会把跟这段内容完全无关的中性场景强行染色。这是全局统计类算法的**结构性缺陷**，
不是调参能解决的，所以才升级到"真实语义分割 + 按区域分层处理"这条路线。回退到整图仿色不是没试过的新思路，
是明确验证过更差的旧方案。

## 真正的根因：同一个文字标签，套住了两种完全不是一回事的东西

拉取用户实测的参考图（商场里明亮的白色吊顶）/目标图（这张照片是偏暗、偏暖的钢架顶棚 + LED 招牌）跑
`analyze_target()`：两者的顶部结构都被 Grounding DINO 标注成 "building"。但实测两边的真实 Lab 统计量：

```
ref  building: l_mean=78.2  mean_ab=[ 1.5,  -6.7]   （很亮，偏冷灰白）
tgt  building: l_mean=48.8  mean_ab=[ 3.7,  12.2]   （中等灰暗，偏暖）
```

ΔL≈29.4，这是一个巨大的差距——"building"这个开放词汇标签，同时框住了"商场白色吊顶"和"钢架结构顶棚"这
两种物理上完全不同的东西。已有的调色逻辑一旦看到标签对上了，就直接把目标图这片区域的统计值往参考图的统计
值上硬拉，medium 档还叠加 1.6 倍"过冲"系数，结果钢架顶棚被拉成了不自然的高亮青蓝色块——跟几乎没变化的
人群（neutral 类，已经很保守）放在一起，视觉上就是"背景跟前景脱节"。

`_class_outlier_confidence`（上一次过冲修复加的）在这里完全不起作用：它只检查"这个像素像不像它自己所属类
别的典型样子"，而这里目标图的每个像素都完美符合目标图自己"building"类的统计特征——问题不在像素层面，在于
"这整个类别跟参考图的同名类别根本不是一回事"这个更高层的判断上。

## 修复：新增"类别配对置信度"，按类别整体外观差距（不做归一化）压制匹配强度

新增 `_class_pair_confidence(ref_stats, tgt_lab, tm)`：

- 直接比较参考图这个类别的真实均值（`l_mean`/`mean_ab`）跟目标图这个类别**自己的**真实均值——用绝对 Lab
  单位算差距，**故意不按任一方类别的内部方差做归一化**（因为一个内部方差本身就很大的类别——比如"building"
  同时框住了亮吊顶和暗结构梁——恰恰说明这个标签不可靠，不该用它的大方差去"容忍"更大的差距）。
- 差距在 15 个 Lab 单位以内：认为"同一种东西、只是光照不同"，正常处理；超过 30：认为"标签蒙对了，东西不是
  一回事"，把该类别的过冲系数压回接近 1.0（只做一次性统计匹配，不外推）。
- 跟 `_class_outlier_confidence`（像素级）相乘后一起控制 `cs`：两者是两种独立的"要不要信任这次匹配"的理由，
  一个看像素，一个看类别整体。
- "neutral"/"skin" 不受这道检查影响，它们各自已经有专门的保守处理（加法 mood-cast / 人脸 hue-lock）。

## 验证

**这张实测图**：钢架顶棚的 `class_confidence=0.04`（几乎完全压制过冲），medium/strong 档渲染后顶棚只是
轻微变亮，跟人群衔接自然，不再是刺眼的青蓝色块。

**20 图回归**：门槛决策完全不变；5 张图有变化，其中 4 张变化很小，**1 张有意外的额外改善**——
`person_event_DSC01549`（户外合影，背景是玻璃幕墙建筑+树）之前背景被过曝匹配到近乎全白模糊一片，看不清
建筑轮廓；修复后玻璃幕墙的纹理、树的绿色都恢复了，说明这不是这一张实测图的偶发问题，是同一类 bug 在这批
回归图里的另一种表现。

**已知仍存在、跟这次改动无关的问题**：`outdoor_sky_058A1908_r2` 里穿浅色衣服的人物有"白色鬼影"现象——这
是此前 Phase A 就记录过的一个安全漏洞的残留表现，改动前后像素几乎没变（本次改动没有触碰这个路径），留作
独立任务。

### C2 训练数据重跑

| 步骤 | 结果 |
|---|---|
| C2.1 导出 | 97/97，`suitable=47`，跟改动前一致 |
| C2.2 拟合 | 97/97，208 class-rows，跟改动前一致 |
| C2.3 训练（ridge，seed=0） | held-out MAE = **3.74**（上一版 4.07 → 这次是**真实改善**，不是噪声波动：predict-mean 基线也从 6.07 降到 5.49，说明训练目标本身变得更一致了——去掉了"同标签、外观差巨大"这种极端离群样本对目标分布的污染） |

结论：这次修复不仅解决了肉眼可见的视觉瑕疵，还让 C2 的训练信号更干净了（基线和 MAE 同时下降，降幅比例
从 33% 提升到 32%左右但绝对数值双降，说明离群样本被移除后整体方差变小），C2.4 Smart Color v2 嫁接可以
放心基于这批数据继续做。
